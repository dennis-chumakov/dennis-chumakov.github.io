Stock Market Direction Prediction

Python
Machine Learning
Financial Data
Can machine learning predict next-day SPY price direction using technical indicators? A comparative study of four classifiers across three iterative feature configurations.
Published

April 17, 2026

Overview

This research paper applied four supervised machine learning classifiers to predict the next-day price direction of the SPY ETF (S&P 500 tracker) using historical price-based technical indicators. The study was conducted across three iterative feature configurations, guided by prior literature and correlation analysis.

NoteResearch Question

How effectively can supervised machine learning classification algorithms predict next-day price direction of the SPY ETF using historical price-based technical indicators?

The dataset contained 4,853 daily OHLCV observations spanning 2005 to 2024, sourced from Kaggle. After preprocessing, 4,804 rows were used for modelling. A chronological 70/30 train/test split was applied, training on data encompassing the 2008 financial crisis, and testing on the COVID-19 shock and post-pandemic inflationary period.

Research Pipeline

The study was structured across three iterations, each refining the feature set based on correlation analysis and model performance:

Iteration Feature Set Key Change
Iteration 1 SMA, RSI, MACD, STD, Volume, Lags Baseline configuration
Iteration 2 + ROC, Williams R, Stochastic, CCI, OBV Expanded with momentum indicators
Iteration 3 Momentum-only, volatility/volume removed Refined after multicollinearity analysis
TipKey Design Decision

A chronological 70/30 split was used rather than random shuffling — random shuffling would introduce look-ahead bias by allowing the model to train on future observations, invalidating the out-of-sample evaluation.

Model Performance Summary

The table below summarises accuracy and AUC-ROC across all three iterations. The naive baseline was 54.4% – the proportion of up days in the dataset.

ImportantKey Finding

Every model performed below the naive baseline of 54.4% across all iterations. AUC-ROC values consistently ranged from 0.48 to 0.51 – near-random classification. This is consistent with weak-form Efficient Market Hypothesis (EMH), suggesting that historical price data alone does not carry sufficient predictive signal for next-day SPY direction.

Feature Engineering

Show feature engineering code 
# Trend indicators
df['SMA_10'] = df['close'].rolling(window=10).mean()
df['SMA_50'] = df['close'].rolling(window=50).mean()
df['SMA_Ratio'] = df['SMA_10'] / df['SMA_50']

# Momentum indicators
exp1 = df['close'].ewm(span=12).mean()
exp2 = df['close'].ewm(span=26).mean()
df['MACD'] = exp1 - exp2
df['ROC_14'] = ta.momentum.ROCIndicator(close=df['close'], window=14).roc()
df['Stochastic_14'] = ta.momentum.StochasticOscillator(
    high=df['high'], low=df['low'], close=df['close'], window=14).stoch()
df['CCI_20'] = ta.trend.CCIIndicator(
    high=df['high'], low=df['low'], close=df['close'], window=20).cci()
df['OBV'] = ta.volume.OnBalanceVolumeIndicator(
    close=df['close'], volume=df['volume']).on_balance_volume()

# Lag features
df['Lag_1_Return'] = df['change_percent'].shift(1)
df['Lag_5_Return'] = df['change_percent'].shift(5)

# Target variable - binary direction label
df['Target'] = (df['close'].shift(-1) > df['close']).astype(int)

# Drop NaN rows
df.dropna(inplace=True)

print(f"Shape after feature engineering: {df.shape}")
print(f"Target distribution:\n{df['Target'].value_counts()}")
print(f"Naive baseline: {df['Target'].mean()*100:.1f}%")

Model Training and Evaluation

Show model training code 
x = df[feature_cols]
y = df['Target']

x_train = x.iloc[:split_index]
x_test = x.iloc[split_index:]
y_train = y.iloc[:split_index]
y_test = y.iloc[split_index:]

scaler = StandardScaler()
x_train_scaled = scaler.fit_transform(x_train)
x_test_scaled = scaler.transform(x_test)

models = {
    'Logistic Regression': LogisticRegression(max_iter=1000, random_state=42),
    'Random Forest': RandomForestClassifier(n_estimators=200, max_depth=5, random_state=42),
    'Gradient Boosting': GradientBoostingClassifier(n_estimators=200, learning_rate=0.05,
                                                     max_depth=3, random_state=42),
    'XGBoost': XGBClassifier(n_estimators=200, learning_rate=0.05, max_depth=3,
                              random_state=42, eval_metric='logloss')
}

model_names, accuracy_scores, precision_scores = [], [], []
recall_scores, f1_scores, roc_auc_scores = [], [], []
confusion_matrices, proba_scores = [], []

for name, model in models.items():
    model.fit(x_train_scaled, y_train)
    y_pred = model.predict(x_test_scaled)
    y_proba = model.predict_proba(x_test_scaled)[:, 1]

    model_names.append(name)
    accuracy_scores.append(round(accuracy_score(y_test, y_pred), 4))
    precision_scores.append(round(precision_score(y_test, y_pred), 4))
    recall_scores.append(round(recall_score(y_test, y_pred), 4))
    f1_scores.append(round(f1_score(y_test, y_pred), 4))
    roc_auc_scores.append(round(roc_auc_score(y_test, y_proba), 4))
    confusion_matrices.append(confusion_matrix(y_test, y_pred))
    proba_scores.append(y_proba)
    print(f"{name} trained successfully")

results_df = pd.DataFrame({
    'Model': model_names,
    'Accuracy': accuracy_scores,
    'Precision': precision_scores,
    'Recall': recall_scores,
    'F1-Score': f1_scores,
    'ROC-AUC': roc_auc_scores
})

print(f"\nNaive baseline: {y_test.mean()*100:.1f}%")
print(results_df.to_string(index=False))

Reflections

This was the most technically demanding project of my degree. The iterative approach taught me that more features do not automatically mean better models and that understanding why a model fails is as valuable as optimising it to succeed.

The near-random results were initially frustrating, but working through the Efficient Market Hypothesis literature reframed them entirely. A model that performs at 51% on SPY next-day prediction is not a failure. It is evidence that the market is doing its job. That conceptual shift, from “my model is broken” to “the market is efficient”, was the most important thing I learned from this project.1
